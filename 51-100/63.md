# **63. Как соотносятся вычислительные платформы и языки описания вычислительного процесса? Как это связано с эффективностью и областью применения?**

Продолжение прошлого билета

Примеры соотношения:

- С -> RISC
- HDL/C -> FPGA
- C/ASM -> DSP
- C/HDL -> ASIP
- HDL -> ASIC

Близость софта к железу (маппинг) - от наибольшего к наименьшому:

- С -> RISC
- HDL/C -> FPGA
- C/ASM -> DSP
- C/HDL -> ASIP
- HDL -> ASIC

Универсальность - от наибольшей к наименьшей:

- С -> RISC
- HDL/C -> FPGA
- C/ASM -> DSP
- C/HDL -> ASIP
- HDL -> ASIC

Эффективность - от наименьшей к наибольшей:

- С -> RISC
- HDL/C -> FPGA
- C/ASM -> DSP
- C/HDL -> ASIP
- HDL -> ASIC

Таким образом, наблюдаем закономерность: чем выше специализация процессора, тем меньше его гибкость в адаптации под новые задачи и ниже уровень абстракции используемых языков. Например: процессоры ASIC используются для расчёта хеш-функций в криптовалютном майнинге. Они показывают высокую эффективность в целевых задачах, но значительно теряют в производительности при смене области применения.
