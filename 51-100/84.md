# **84. Как можно оптимизировать инструкции при помощи микропрограммирования? Приведите пример.**

Мы можем сочинить или изменить существющие команды так, чтобы они работали на 17% быстрее. \*Пенской

Благодаря этому мы можем делать разные реализации поставленной задачи, пример вот из прошлых билетов, очв что выгоднее нам.

Вот эта команда съест 5 тактов (вроде)

```
add #01 <- 34 + #03      5t
```

Вот эти съедят 8 тактов на троих (вроде)

```
acc <- 34                2t
acc <-+ #03              3t
#01 <- acc               3t
```
