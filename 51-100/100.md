# **100. Каковы принципы конвейерного исполнения инструкций? Как это влияет на загрузку и производительность?**

Придя к инструкциям одинакового размера и получив возможность загружать инструкцию за такт, мы можем задуматься о выполнении инструкций не последовательно, паралельно по шагам выполнения инструкции (fetch, decode, execute, memory, write)

`Конвейеризация` - это метод выполнения инструкций, при котором различные этапы обработки команды выполняются паралельно для разных инструкций в последовательности

Принцип:

- Выполнение каждой инструкции можно разбить на несколько конкретных этапов, которые задействуют непересекающиеся части процессора и выполняются за один такт (к билету у морской артилерии)
- Этапы независимы
- В результате чего инструкции выполняются не последовательно, а практически паралельно:

```
ins1      FETCH  ->  DECODE  ->  EXECUTE  ->  MEMORY  ->  WRITE
ins2                 FETCH   ->  DECODE   ->  EXECUTE ->  MEMORY ->  ...
```

Очев делая проц таким образом мы крайне бустим производительность и уменьшаем время простоя, с другой стороны начинаем сталкиваться с характерными проблемами паралельности.
