# **11. Как можно рассматривать систему с точки зрения структуры? Каковы причины множественности структуры?**

Система -> подсистемы -> части

Система - это комбинация взаимодействующих между собой элементов, необходимых для покрытия технических требований.
Смотря на систему нам необходимо видеть основные её части и как они взаимодествют между собой.
Пример: бэкэнд на java spring с модулями и java ee + взаймодейтсвие с фронтендом на javascript

Причины множественности структур. Одну и ту же систему можно рассматривать с разных сторон:

- Уровень абстракций (ничего не мешает рассматривать вашу систему с того как биты катаюся по шины, естественно по реалу такой хуйней никто заниматься не будет)
- Целей анализа (разные заинтресованные стороны например)
- Масштабируемости и гибкости
- Технологического стека

Итог по структурному подходу:

- Декомпозирование сложных систем на управляемые части
- Оптимизация взаимодействия компонентов
