# **36. Каковы особенности реализации "условного оператора" в комбинационных схемах?**

Уловный оператор реализован через мультиплексор. Специальное логическое устройство - мультиплексор (MUX) (выбирает, какой результат вывести, основываясь на условном сигнале).

```
I0 ---------\
    |        \
    |         |------ out
    |        /
I1 ---------/
       ^
       |
      sel
```

Реализация на Питончике:

```
in0 = f0(...)
in1 = f1(...)

out = in0 if sel else in1
```

Реализация на Verilog:

```
assign in0 = f0(...);
assign in1 = f1(...);

assign out = sel ? in0 : in1;
```

Как это выглядит:

1. На вход схемы подаются данные.
2. Физический ток одновременно пошел по двум разным кремниевым трактам: один блок вентилей считает Функцию_1, а второй — Функцию_2.
3. На выходах обоих блоков появляются готовые результаты.
4. В самом конце ставится мультиплексор (MUX) - тот самый элемент, схему которого мы только что рисовали. Он работает как железнодорожная стрелка. На его управляющий вход (sel) подается результат проверки условия (if).
5. Мультиплексор просто пропускает на выход out один из уже посчитанных результатов, а второй - просто отбрасывает.
