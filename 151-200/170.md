# **170. Каковы особенности реализации кеша с прямым отображением?**

`Кеш с прямым отображением` (`Direct-Mapped Cache`) - это метод организации кеш-памяти, при котором каждому блоку основной памяти соответствует строго определённая линия кеша

Преимущества:

- Существенно уменьшается объем логики;
- Снижается потребление энергии;
- Уменьшается объем памяти, необходимый для хранения тегов;
- Реализация становится ближе к механике хэш-таблиц: в качестве «хэша» используется часть адреса, по которой выбирается единственная подходящая кеш-линия.

Ограничения:

- Невозможно держать одновременно в кеше блоки, которые отображаются в одну и ту же линию
- Это может провоцировать частые кеш-промахи, особенно при доступе к смежным областям памяти
- Не подходит для некоторых типов нагрузок (например, итерации по массиву с шагом, равным размеру кеша)

Принцип работы следующий. Сначала адрес разбивается на три части:

- Offset (младшие биты) - смещение внутри строки;
- Index (средние биты) - указывает, в какую единственную кеш-линию должен попасть блок;
- Tag (старшие биты) - хранится в кеш-строке и проверяется при обращении.

При доступе к кешу:

- Сначала по Index извлекается одна конкретная строка из кеша;
- Затем Tag сравнивается с тем, что хранится в этой строке;
- Если совпало - кеш-попадание. Если нет - промах.
