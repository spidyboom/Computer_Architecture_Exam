# **182. Как происходит обмен информацией о кеш-линиях через отслеживание? Как этот подход масштабируется?**

В случае `отслеживания` (`Snooping`) каждый кеш подключается к общей шине наблюдения, транслирует на нее все запросы обращения к памяти и отслеживает изменения своих данных другими кешами. При модификации ядром данных, оно расслыает сигнал по шине, все остальные кеши слушают эти сигналы и, если имеется копия этих данных, сразу помечают свою кеш-линию как не актуальную (`Invalid`).

Преимущества:

- Высокая скорость. Нет задержек перед чтением и записью данных.
- Простота реализации. Не требуется отдельной сложной структуры, как при справочнике.

Недостатки:

- Ограниченная масштабируемость. Для корректной работы каждому кешу нужно иметь число входов равное числу других кешей, из-за этого быстро растет площадь процессора (квадратично).
- При высокой активности ядер, шина может оказаться перегружена, что может затормозить работу процессора.
