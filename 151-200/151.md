# **151. Какие существуют способы интеграции и взаимодействия контроллера прямого доступа к памяти (DMA) с процессором?**

DMA имеет доступ к памяти наравне с процессором занимая естественным образом шину данных. Отсюда проблема конфликта, когда оба хотят получить доступ к памяти.

Способы взаимодействия:

- `Пакетный режим`. Даем приоритет для DMA: пока обрабатывается очередной пакет, процессор ждет.
- `Циклический режим`. Работаем по циклам, в каждом из которых даем волю отдельно DMA, отдельно процессору
- `Прозрачный режим`. Процессор имеет приоритет по доступу к памяти над DMA. DMA работает по остаточному принципу.

Способы интеграции:

- `Third-party`: Управление DMA осуществляется процессором(есть специальное отдельное устройство, которое висит на шине рядом с памятью и мы с ним можем взаимодействовать. НО в таком случае контроллер dma тупеньким становится, он не может обрабатывать прерывания, может только переносить данные)
- `Bus mastering`: Управление DMA может осуществляться и устройствами ввода-вывода (на шину ставим ставим самостоятельное устройство, которое имеет свою систему прерываний, информировать процессор о готовности данных к чтению только после прочтения всех данных. Карочи весь ввод-вывод делегировать на DMA устройство)
