# **180. Что такое когерентность кеш-памяти и каково её значение для многоядерных систем? Какие существуют возможные состояния кеш-линий?**

`Когерентность` - целостность данных в кешах процессора между собой и в памяти. Т.е. ситуация, когда все ядра при чтении будет получать одинаковые и более того актуальные данные, согласованные с RAM

В многоядерных системах очевидно проблемы с чтением, поддерживанием актуальных данных.

Вводим состояния кэшлиний `MOESI`:

- `Modified` - текущее ядро изменило эту кеш линию, здесь свежие данные и они есть ТОЛЬКО в этом ядре
- `Owned` - текущее ядро явяяется владельцем ИЗМЕНЕННЫХ данных и готово выдать другим ядрам корректные данные, другие ядра могут хранить данные в состоянии `Shared`. Ядро `Owned` берет обязанность по доставки корректных данных в RAM
- `Exclusive` - ядро взяло данные из RAM. Они находятся ТОЛЬКО у него. Оно может их менять.
- `Shared` - данные актуальны, но владеют ими несколько ядер, от чего никто из них не может их менять. Чтобы одно ядро могло менять данные, нужно инвалидировать эти данные у всех остальных ядер
- `Invalid` - данные невалидны или пустая строка

Три основных механизма обеспечения когерентности кешей:

- справочник (Directory)
- отслеживание (Snooping)
- перехват (Snarfing)
