# **126. Как реализуется кооперативная многозадачность на уровне приложения через Event-loop?**

Одна из двух возможных реализаций кооперативной многозадчночсти.

Есть три элемента:

- Очередь запросов/событий: Сюда поступают входящие события (кликнули мышкой, прилетел сетевой пакет, таймер отсчитал секунду).

- Event Loop: Бесконечный цикл. Его единственная задача - вытащить событие из очереди и вызвать привязанную к нему функцию-обработчик (Call-back).

- Асинхронное окружение: Это тяжелые операции, выполнение которых в основном потоке привело бы к зависанию.
