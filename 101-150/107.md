# **107. Какие существуют способы разрешения конфликтов по управлению во время компиляции?**

Конфликты заключаются в том, что мы на N инструкции переходим N +- P инструкций впред/назад, а в конвейер пытается загрузиться N + 1 интрукция.

1. Оптимизация предсказателя переходов (в случае работы с условными переходами)
2. Уменьшение числа переходов:
   - Уменьшение кол-ва циклов, путем его увеличения. Пример с лекции:

```
for (int x = 0; x < 100; x++) {
  delete(x);
}

for (int x = 0; x < 100; x+=5) {
  delete(x);
  delete(x + 1);
  delete(x + 2);
  delete(x + 3);
  delete(x + 4);
}
```

    - Условное перемещения данных. Вместо того чтобы городить развилку if-else с командами условного перехода, компилятор генерирует последовательный код, но использует специальные условные макрокоманды (например, CMOV — Conditional Move в x86, или условное выполнение инструкций в ARM).
