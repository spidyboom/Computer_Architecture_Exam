# **115. Как реализуется ввод-вывод с точки зрения системы команд в Memory Mapped IO?**

`Memory-Mapped-IO` (`MMIO`) - способ организации обмена в ВУ, при котором в памяти у нас есть отдельные адреса, замапленные под работу с внешними устройствами. Иными словами, ВУ общаетсмя с нами через память, а не через отдельные адреса как в PMIO.

1. Регистры ВУ отображаются в адресное пространство памяти.
2. Ввод-вывод реализуется через инструкция доступа к памяти.

Достоинтства:

1. Простота процессора (не меняем микрокод)
2. Единый набор механизмов доступа: автоикремент, векторные операции, работа с барьерами
3. Обработка данных без переноса в память
4. Адресное пространство памяти.

Недостаткти:

1. Одна шина для ввода-вывода и памяти. Разница в скорости шин.
2. Неоднородность памяти, сложная конфигурация системы.
3. Избыточный адрес для УВвода-Вывода
4. Проблемы с параллелизмом уровня инструкций и кешами:
   - flush При вводе-выводе
   - порядка записи регистров
